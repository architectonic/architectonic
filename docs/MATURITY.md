# Maturity

Architectonic 0.2 is experimental and pre-1.0.

## What is implemented

- reproducible git and npm layer installation;
- explicit full and partial profiles;
- machine-readable workspace and package manifests;
- canonical-entry mapping through `architectonic map`;
- installation checks through `architectonic doctor`;
- semantic protocol conformance through `architectonic verify`;
- local-change detection and guarded upgrades;
- a reference path for creating installed agents from agents-layer specifications;
- source tests and packed-artifact verification before npm publication.

## What is not yet established

- universal productivity improvement;
- universal reduction in agent errors;
- independent benchmark replication;
- compatibility guarantees associated with a 1.0 release;
- safe unattended mutation of private organizations or knowledge corpora;
- certification for legal, financial, medical, security, or other regulated decisions.

## Evidence standard

A mechanism is considered demonstrated when its implementation, fixture, test, and expected result are public and reproducible. A performance claim requires paired conditions, raw runs, a scoring method, model and provider metadata, and stated limitations.
